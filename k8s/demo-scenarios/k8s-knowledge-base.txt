Q: Kubernetes cluster ka architecture explain karo
Kubernetes architecture do parts mein divide hota hai:
    Control Plane â†’ cluster ke decisions yahan liye jaate hain (e.g., scheduling, state management)
    Worker Nodes â†’ yahan pe actual pods chalte hain

Control Plane Components:
    kube-apiserver â€” cluster ka entry point
    etcd â€” cluster ka database
    scheduler â€” pods ko nodes pe assign karta hai
    controller-manager â€” state monitoring karta hai

Worker Node Components:
    kubelet â€” instructions ko execute karta hai
    kube-proxy â€” networking rules manage karta hai
    container runtime â€” like Docker, containers ko chalata hai
